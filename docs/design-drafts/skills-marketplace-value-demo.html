Scenario 3 / After a failure, feedback goes back to the creator
User
This report Skill started writing without asking about the audience.

Agent
That is useful Skill feedback. I will send the reproduction context to the author so the next version can add a rule to confirm the audience first.

Bash(agent-skills feedback @demo/report-writer \
  --type issue \
  --agent codex \
  --message "Report writer should ask for audience before drafting" \
  --context "User asked for a project report. The Skill started drafting without confirming audience or purpose.")
╭────────────────────────────────────────╮
│ ✓ Feedback submitted #1                │
│ Skill: @demo/report-writer v0.1.0      │
╰────────────────────────────────────────╯

Agent
Feedback submitted. The creator can see this real failure example in the Web detail page or My Skills, then update the Skill.
