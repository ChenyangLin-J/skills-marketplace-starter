# Report Writer

Use this Skill when the user wants to turn rough findings into a clear report.

## Workflow

1. Ask what audience the report is for.
2. Ask what decision the report should support.
3. Separate facts, interpretation, and recommendations.
4. Put the conclusion first.
5. Keep assumptions and data gaps visible.

## Report Shape

```markdown
# Title

## Summary

## Key Findings

## Evidence

## Recommendation

## Open Questions
```

## Quality Bar

- A busy reader should understand the conclusion in the first minute.
- Every recommendation should tie back to evidence.
- Uncertainty should be explicit.
